Agenda - Day 1

  1. Intro to Apache Spark using spark-shell
  2. My first Spark SQL App
    • IntelliJ IDEA / sbt / spark-submit
  3. Deploying Spark app to Spark Standalone
  4. Standard Functions / Operators / UDFs in Spark SQL
  5. RDDs, DataFrames, Datasets

Agenda - Day 2 (1 of 2)

  1. DataSource API
    • Reading and saving datasets
    • files (Parquet), JDBC (PostgreSQL), NoSQL (Cassandra)
  2. Encoders
  3. Spark in Cluster Mode (Spark Standalone, Hadoop Yarn and Apache Mesos)
  4. Deploying to a Cluster - spark-submit
  5. Client, Driver, Master, Executors, Workers, Jobs, DAG, Stages, Tasks

Agenda - Day 2 (2 of 2)

  1. Performance tuning
    • Reading/viewing/understanding the Web UI
    • Spark Listeners
    • Spark History Server
  2. Monitoring execution from within the application, i.e. how to discover that a job has failed, or takes too long

Agenda - Day 3

  1. Spark SQL
    • Aggregation
    • Windows
  2. Structured Query Plan and Optimizations
  3. Spark Structured Streaming
  4. Spark MLlib
  5. Thrift JDBC/ODBC Server
  6. Dynamic Allocation of Executors

Agenda - Day 4 (1 of 2)

  1. Broadcast Variables and Accumulators
  2. Docker support
    • Docker with the Master
    • Docker with a Client, and then n clients
    • How to set it up
  3. Caching, Cache manager and Persistence

Agenda - Day 4 (2 of 2)

  1. Sharing Spark Master with other jobs, long running jobs (like an API) and short lived jobs at the same time, best way to solve this
  2. Spark Streaming / Apache Kafka 0.10+
    • Introduction to Spark Streaming
    • Kafka Architecture / Setup
    • StreamingContext and Operators
    • Kafka Direct Approach (No Receivers)

Prerequisities

  1. Some programming experience using modern programming language (preferably on JVM)
    • Java, Python, Scala, C#
  2. Installed
    • Java SE 8
    • IntelliJ IDEA
  3. Downloaded
    • Apache Spark
    • sbt
    • PostgreSQL
    • Kafka
  4. Willingness to ask PLENTY of questions
